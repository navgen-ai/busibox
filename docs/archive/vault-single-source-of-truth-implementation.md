# Vault as Single Source of Truth - Implementation Summary

## What Changed

Busibox now uses **Ansible Vault as the single source of truth** for all secrets across all environments (local Docker, staging, production).

The `.env.local` file for Docker Compose is now **auto-generated** from the vault, eliminating redundancy and ensuring consistency.

## Key Benefits

✅ **Single source of truth** - All secrets in one place (vault.yml)
✅ **No duplication** - Secrets aren't maintained in multiple files
✅ **Consistent** - Same secrets used for local and remote deployments
✅ **Secure** - Vault is encrypted, `.env.local` is ephemeral
✅ **Simple** - Edit vault → Generate `.env.local` → Done

## New Commands

```bash
# Generate .env.local from vault (use this regularly)
make vault-generate-env

# Migrate existing .env.local to vault (one-time)
make vault-migrate

# Sync vault structure with example (after updates)
make vault-sync
```

## Implementation Details

### 1. Migration Script

**`scripts/vault/migrate-env-to-vault.sh`**

- Reads existing `.env.local` file
- Decrypts current vault (or uses `vault.example.yml` as base)
- Maps `.env.local` values to vault structure using Python/PyYAML
- Merges values intelligently
- Creates timestamped backups
- Encrypts and saves new vault

**Mapping:**
```
.env.local                    → vault.yml structure
─────────────────────────────────────────────────
POSTGRES_PASSWORD             → secrets.postgresql.password
MINIO_ACCESS_KEY              → secrets.minio.root_user
MINIO_SECRET_KEY              → secrets.minio.root_password
OPENAI_API_KEY                → secrets.openai.api_key
BEDROCK_API_KEY               → secrets.bedrock.api_key
LITELLM_MASTER_KEY            → secrets.litellm_api_key
AUTHZ_ADMIN_TOKEN             → secrets.authz.admin_token
AUTHZ_MASTER_KEY              → secrets.authz.master_key
BETTER_AUTH_SECRET            → secrets.better_auth_secret
SSO_JWT_SECRET                → secrets.jwt_secret
ADMIN_EMAIL                   → secrets.admin_email
ALLOWED_EMAIL_DOMAINS         → secrets.allowed_email_domains
GITHUB_AUTH_TOKEN             → secrets.github_token
AGENT_MANAGER_CLIENT_ID       → secrets.agent-manager.admin_client_id
AGENT_MANAGER_CLIENT_SECRET   → secrets.agent-manager.admin_client_secret
```

### 2. Generate Script

**`scripts/vault/generate-env-from-vault.sh`**

- Decrypts vault
- Extracts secrets using Python/PyYAML
- Generates `.env.local` with proper format and comments
- Adds header indicating auto-generation
- Includes development defaults (POSTGRES_USER, LITELLM_BASE_URL, etc.)
- Overwrites existing `.env.local`

**Output format:**
```bash
# =============================================================================
# Busibox Local Development Environment Configuration
# =============================================================================
#
# AUTO-GENERATED from Ansible vault
# Generated: 2026-01-18 19:43:46
#
# DO NOT EDIT THIS FILE MANUALLY!
# Edit the vault instead: ansible-vault edit provision/ansible/roles/secrets/vars/vault.yml
# Then regenerate: bash scripts/vault/generate-env-from-vault.sh
#
# =============================================================================

# ... actual env vars ...
```

### 3. Vault Sync Script (Updated)

**`scripts/vault/sync-vault.sh`**

Already existed, but now creates timestamped backups:
- `vault.backup.YYYYMMDD-HHMMSS.yml` (before each sync)
- `vault.removed.YYYYMMDD-HHMMSS.yml` (unmapped secrets)

### 4. Makefile Targets

Added new section to `Makefile`:

```makefile
# ============================================================================
# VAULT & ENV MANAGEMENT
# ============================================================================
vault-generate-env:
	@bash scripts/vault/generate-env-from-vault.sh

vault-migrate:
	@bash scripts/vault/migrate-env-to-vault.sh

vault-sync:
	@bash scripts/vault/sync-vault.sh
```

Updated help output to show these commands.

### 5. Vault Password File

Created `~/.vault_pass` for local development:

- Used automatically by scripts
- `chmod 600` for security
- Not in git (personal file)

### 6. Docker Integration

`docker-compose.local.yml` already uses `.env.local`:


No changes needed to Docker Compose - it continues to work as before, but now `.env.local` is generated from the vault.

## Technical Details

### Dependencies

- **Python 3:** For YAML parsing
- **PyYAML:** `pip3 install --user --break-system-packages pyyaml`
- **ansible-vault:** Already required by Busibox

### Bash Compatibility

All scripts use `set -euo pipefail` and are compatible with:
- macOS bash 3.2+
- Linux bash 4.0+

### Error Handling

Scripts include:
- `set -euo pipefail` for strict error handling
- Cleanup traps for temp files
- Backup before changes
- Rollback on failure
- User confirmation for destructive operations

### Python Integration

Embedded Python scripts in bash for robust YAML handling:
- Flattening/unflattening nested structures
- Preserving YAML ordering
- Handling special characters
- Base64 decoding for AWS credentials
